# OOAD Checkpoint Draft Plan

Audience: CSC0005 checkpoint presentation audience and instructor.

Objective: Present the project as a rule-driven investment analysis platform rather than a CRUD-heavy stock query site.

Narrative arc:
1. Start with the upgraded project identity.
2. Show that teacher feedback was incorporated.
3. Summarize progress and work breakdown.
4. Cover the required diagrams in a clear order.
5. End with implemented results and next steps.

Slide list:
1. Cover
2. Revision based on teacher feedback
3. Progress summary and team work breakdown
4. Use case diagram
5. Activity diagram
6. Architecture diagram
7. Data storage
8. UI mockups
9. Class diagram A - Portfolio Domain
10. Class diagram B - Alert and Backtest Domains
11. Design Pattern 1 - Strategy Pattern
12. Design Pattern 2 - Factory Pattern
13. Conclusion and next steps

Source plan:
- Use the OOAD checkpoint instructions from OOAD.P2.pdf.
- Use the current implemented project structure in the workspace.
- Use placeholder member names where the exact team composition is unknown.

Visual system:
- Warm light background
- Dark ink text
- Mint, blue, gold, and coral accents
- Rounded cards and diagram blocks

Editability plan:
- All titles, bullets, cards, and diagrams are editable PowerPoint text and shapes.
- No rasterized screenshots are required for this draft version.
