# OOAD 期末 Demo 操作步驟與口語講稿

**目標長度：** 8 到 10 分鐘  
**展示方式：** 使用手機或 Chrome DevTools mobile viewport 展示  
**Demo 主軸：** 讓操作和口白一一對應，重點放在「目前畫面正在完成哪個 use case」。

## 錄製前準備

1. 啟動系統。

   ```bash
   cd frontend
   npm run dev
   ```

2. 開啟 `http://localhost:3000`。
3. 使用 Chrome DevTools 切換 mobile viewport，例如 iPhone 14 Pro 或 Pixel 7。
4. 準備一組測試帳號。建議先準備好已註冊帳號，demo 時可以簡短展示註冊入口，不一定真的註冊一次。
5. 先確認可查股票代號，例如 `2330`、`0050`，或資料庫裡確定有歷史價格的股票。
6. 先準備一個 portfolio，或 demo 時現場建立一個名稱簡短的 portfolio。
7. 管理員市場資料同步可能較久，demo 時只展示畫面與選項，不建議真的跑完整同步。

## 完整操作步驟與講稿

### 1. 開場與手機畫面

**操作：**

1. 畫面停在首頁或登入頁。
2. 確認畫面是手機尺寸。
3. 可以先慢慢滑一下頁面，讓老師看到這是 mobile browser 操作。

**講稿：**

接下來示範我們期末專題的實際系統操作。這個系統是「股市投資組合、警示與策略回測分析平台」。

因為期末要求需要在 mobile platform 上展示，所以我現在使用手機版瀏覽器的畫面來操作。接下來會按照一般使用者的流程，從註冊和登入開始，接著查詢股票、加入自選股、管理投資組合、設定警示、查看通知，最後執行策略回測。

Demo 過程中我會直接說明目前畫面對應到哪一個 use case。

### 2. 註冊入口、登入與 Dashboard

**操作：**

1. 停在 Login 頁。
2. 指出「沒有帳號可以先註冊」的入口。
3. 點到 Register 頁，展示註冊欄位：使用者名稱、email、password。
4. 不一定真的送出註冊；展示完後回到 Login。
5. 輸入測試帳號與密碼登入。
6. 進入 Dashboard。
7. 稍微往下滑，展示市場概況、投資組合摘要、常用功能入口。

**講稿：**

首先是使用者進入系統的流程。現在畫面是在登入頁，如果使用者已經有帳號，可以直接輸入 email 和密碼登入。

如果還沒有帳號，可以先點這裡到註冊頁。註冊頁會要求使用者填寫基本資料，例如使用者名稱、email 和密碼。註冊完成後，就可以回到登入頁使用這組帳號登入。

這邊 demo 我使用事先準備好的測試帳號登入，避免現場註冊花太多時間。

登入後進入 Dashboard。Dashboard 是使用者進入系統後第一個看到的總覽頁。這裡可以看到市場相關資訊、投資組合摘要，以及一些常用功能入口。

這一段主要對應到「使用者進入系統」和「查看總覽」的流程。它的作用是讓使用者不用一進來就進入細節頁，而是先看到目前整體狀態。

### 3. 股票查詢與個股頁

**操作：**

1. 從 Dashboard 或側邊欄/導覽列進入 Stocks 頁。
2. 說明股票可以在 Stocks 頁查，也可以從某些頁面的搜尋入口查。
3. 在搜尋框輸入 `2330`。
4. 等搜尋結果出現後，點擊股票進入個股頁。
5. 停在個股頁頂部，展示股票名稱、代號、最新價格、漲跌幅等摘要。
6. 往下滑到圖表區，展示歷史走勢圖。
7. 如果頁面有區間或圖表切換，示範切換一次。
8. 往下滑到財務或其他資訊區，展示財報/指標/ETF 相關資料。
9. 指出個股頁上的延伸操作入口，例如加入自選股、建立警示、回測或交易按鈕。

**講稿：**

接下來是第一個主要 use case：查詢股票資訊。

使用者可以從 Dashboard 的入口進來，也可以從側邊欄或導覽列點到 Stocks 頁。進入 Stocks 頁後，這裡有搜尋框，可以輸入股票代號或股票名稱。

我這裡輸入 `2330` 作為範例。搜尋結果出現後，點進個股頁。

現在進到個股頁最上方，這裡會先顯示使用者最需要快速看到的資訊，例如股票代號、公司名稱、最新價格，以及今天的漲跌狀況。這一段讓使用者先判斷目前這檔股票的大致狀態。

接著往下看，可以看到價格走勢圖。這裡可以觀察股票在不同時間區間的價格變化。如果要看更詳細的日內或歷史走勢，也可以切換圖表區間或圖表類型。

再往下，頁面會顯示更進一步的資料，例如財務指標、歷史價格或其他市場資訊。這些資料可以幫助使用者不只看股價，也能看公司基本面或歷史表現。

個股頁不只是查詢資訊，它也是其他功能的入口。使用者看完股票後，可以直接把它加入自選股，也可以針對這檔股票建立警示，或拿它去做策略回測。這樣就把「查詢股票資訊」這個 use case，接到後面的投資管理流程。

### 4. 自選股與投資組合

**操作：**

1. 在個股頁點擊加入自選股，或回到 Stocks 頁從列表加入。
2. 進入 Watchlist 頁。
3. 展示剛加入的股票與清單資訊。
4. 進入 Portfolios 頁。
5. 如果沒有 portfolio，點建立投資組合，輸入名稱，例如 `Demo Portfolio`。
6. 進入 portfolio 明細頁。
7. 點擊新增交易或交易按鈕。
8. 選擇買進，輸入股票代號 `2330`、數量、價格。
9. 送出後，停在持股列表或摘要區，展示數量、成本、市值、損益。
10. 若時間夠，新增賣出交易，故意輸入超過持股的數量，展示錯誤訊息。

**講稿：**

看完個股資料後，下一步通常是把有興趣的股票加入自選股。這裡我點擊加入自選股。

接著進入 Watchlist 頁。這個頁面對應到「管理自選股」use case。使用者可以在這裡集中追蹤自己關心的股票，之後就不用每次重新搜尋。

再來進入 Portfolios 頁。這裡對應到「管理投資組合」use case。使用者可以建立多個投資組合，例如長期投資、短期交易，或不同策略用的組合。

如果目前沒有投資組合，可以先新增一個。這裡我建立一個 demo 用的 portfolio。建立後進入明細頁，可以看到這個投資組合目前的摘要和持股。

接下來示範「記錄買進交易」。我點新增交易，選擇買進，輸入股票代號、數量和價格。送出後，系統會把這筆交易加入投資組合，並更新持股資訊。

現在畫面上可以看到目前持有的股票、數量、成本、市值和損益。這代表系統不只是存一筆交易，而是會把交易整理成目前的持股狀態。

如果要示範賣出，也可以新增一筆賣出交易。系統會檢查目前持股是否足夠。如果我輸入超過持股的賣出數量，系統會顯示錯誤，避免記錄不合理的交易。這一段對應到「記錄賣出交易並驗證庫存是否足夠」這個 use case。

### 5. 警示與通知

**操作：**

1. 從側邊欄/導覽列進入 Alerts 頁。
2. 指出上方建立警示表單。
3. 輸入股票代號 `2330`。
4. 選擇規則類型，例如「價格高於」或「單日漲幅至少」。
5. 輸入門檻值。
6. 送出建立規則。
7. 停在規則列表，展示股票、條件、目前價格、啟用狀態。
8. 切換啟用 / 停用一次。
9. 點擊手動評估或重新整理通知。
10. 往下看通知列表，展示通知內容與標記已讀。

**講稿：**

接下來進入 Alerts 頁，示範警示和通知流程。

這個頁面上方是建立警示規則的表單。使用者先輸入股票代號，例如 `2330`，再選擇規則類型。系統支援幾種常見條件，例如價格高於某個數字、價格低於某個數字，或單日漲跌幅達到某個百分比。

這裡我建立一個示範用的警示，設定好門檻值後送出。

建立完成後，警示會出現在列表裡。列表中可以看到股票代號、公司名稱、警示條件、目前價格，以及這個規則是否啟用。

如果使用者暫時不想收到某個提醒，可以直接把規則停用。之後如果又需要，也可以再啟用。這一段對應到「建立警示規則」和「啟用 / 停用警示規則」兩個 use cases。

下面是通知區。當警示條件被觸發時，系統會建立通知。使用者可以在這裡查看通知內容，也可以把通知標記成已讀。這一段對應到「查看通知與標記已讀」use case。

### 6. 策略回測與結果

**操作：**

1. 從側邊欄/導覽列進入 Backtests 頁。
2. 停在表單上方，說明這裡輸入回測條件。
3. 輸入股票代號 `2330`。
4. 選擇開始日期與結束日期，例如最近一年或最近兩年。
5. 選擇策略，例如 Moving Average Cross。
6. 說明不同策略可切換，但 demo 先用均線策略。
7. 輸入短均線 `5`、長均線 `20`、初始資金 `100000`。
8. 若畫面有手續費、交易稅、滑價欄位，簡短帶過。
9. 按下執行回測。
10. 等待結果產生。
11. 停在績效摘要，逐一說明總報酬、最大回撤、勝率、交易次數。
12. 往下滑到 equity curve 圖。
13. 往下滑到交易明細表。
14. 如果已有多筆結果，勾選兩筆做比較，展示比較曲線。

**講稿：**

接下來示範策略回測。這一段對應到「執行策略回測」和「查看回測結果」use case。

進入 Backtests 頁後，上方是回測表單。使用者要先輸入股票代號，這裡我一樣使用 `2330`。接著選擇回測期間，例如最近一年或最近兩年。

再來選擇策略。系統支援多種策略，demo 這裡先選最容易理解的均線交叉策略。均線交叉策略會使用短期均線和長期均線做判斷，所以這裡我設定短均線為 5，長均線為 20。

接著設定初始資金，例如 100000。下面如果有手續費、交易稅或滑價設定，可以依照預設值，也可以讓使用者自行調整，讓回測更接近實際交易。

設定完成後，按下執行回測。系統會根據指定期間的歷史價格，模擬這個策略如果真的交易，最後會得到什麼結果。

回測完成後，畫面上會先看到績效摘要。這裡包含最後資產、總報酬率、最大回撤、勝率和交易次數。總報酬率可以看策略最後有沒有賺錢；最大回撤可以看過程中最大虧損壓力；勝率和交易次數則可以看策略交易頻率和成功比例。

往下滑可以看到資產曲線，也就是 equity curve。這張圖可以看出資產是穩定上升，還是中間有很大的波動。

再往下是交易明細，會列出回測過程中模擬的買進和賣出紀錄。這讓使用者可以檢查策略是在什麼時間點進出場。

如果已經做過多次回測，也可以勾選不同結果進行比較。例如同一檔股票比較不同策略，或同一個策略比較不同參數。畫面會把不同回測的曲線放在一起，方便使用者判斷哪一組結果比較符合自己的需求。

### 7. 管理員市場資料維護

**操作：**

1. 進入 Admin Market Data 頁。
2. 停在資料狀態區，展示各資料表筆數與最新日期。
3. 往下滑到品質檢查區，展示檢查結果。
4. 展示同步來源選項，例如 Auto、FinLab、Free、Public Incremental。
5. 不要真的跑完整同步，說明原因。

**講稿：**

最後展示管理員市場資料維護功能。這一段對應到「更新市場資料」use case。

因為前面所有功能，包含股票查詢、警示和回測，都需要市場資料，所以系統提供管理員頁面查看資料狀態。

在這裡可以看到不同資料表目前的資料量，以及最新的價格日期。這可以幫助管理員確認資料是不是太舊，或某些資料表是不是沒有成功匯入。

往下看可以看到資料品質檢查，例如是否有股票缺少價格、資料日期落後，或基本資料不完整。

同步來源這裡提供不同模式，例如 Auto、FinLab、Free source 和 Public Incremental。Demo 時我不實際執行完整同步，因為市場資料同步可能會花比較久，而且會受到外部資料來源速度影響。不過從畫面可以看到，系統已經提供資料維護和同步紀錄的管理流程。

### 8. 總結

**操作：**

1. 回到 Dashboard，或停在 Backtests 結果頁。
2. 用 20 到 30 秒收尾。

**講稿：**

以上就是系統的完整 demo。

剛剛依序展示了使用者註冊與登入、查詢股票資訊、管理自選股、管理投資組合、記錄買進和賣出交易、建立警示規則、查看通知、執行策略回測、查看回測結果，以及管理員更新市場資料。

整體來說，這個系統不是只有單純查股票，而是把投資者常見的操作流程串起來。使用者可以先查詢標的，再加入自選股或投資組合，接著設定提醒，最後用回測功能檢查策略在歷史資料中的表現。

這些畫面也對應到我們報告中的主要 use cases，證明系統目前的功能已經可以支援完整的投資分析流程。

## Demo 備案

1. 股票查詢失敗時，改用已確認有資料的股票代號。
2. 註冊流程不穩時，只展示註冊頁欄位，登入仍使用預先準備帳號。
3. 回測執行太久時，展示已建立的回測紀錄。
4. 市場資料同步不要現場執行完整流程，只展示設定與狀態。
5. 警示不容易即時觸發時，可以建立接近目前價格的門檻，或展示已有通知。
6. 若實體手機連不到 localhost，改用 Chrome DevTools mobile viewport。

## 建議 Demo 資料

| 項目 | 建議值 |
| --- | --- |
| 股票代號 | `2330`、`0050`，或資料庫中有完整歷史價格的標的 |
| Portfolio 名稱 | `Demo Portfolio` |
| 買進交易 | `2330`，數量 `10`，價格使用接近目前價格的數字 |
| 賣出錯誤示範 | 賣出數量大於目前持股 |
| 回測期間 | 最近一年或最近兩年 |
| 初始資金 | `100000` |
| Moving Average | short window `5`，long window `20` |
| RSI | window `14`，oversold `30`，overbought `70` |
| Breakout | lookback window `20` |
| 警示條件 | 價格高於目前價附近，或單日漲跌幅至少 1% |
